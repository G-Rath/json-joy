Module json-crdt-patch

The json-crdt-patch sub-library implements JSON CRDT Patch specification, which is a way to describe patches (changes) to JSON CRDT documents.

This sub-library contains contains:

  • Classes for all JSON CRDT Patch operations.
  • Patch class, which represents a JSON CRDT Patch.
  • PatchBuilder class, which is used to build a JSON CRDT Patch.
  • json-crdt-patch/clock module implements the logical clock.
  • json-crdt-patch/codec/verbose module implements "verbose" serialization codec.
  • json-crdt-patch/codec/compact module implements "compact" serialization codec.
  • json-crdt-patch/codec/binary module implements "binary" serialization codec.

Usage

The most common way to create a Patch is to use PatchBuilder class.

Instantiating a PatchBuilder

Create a new instance of PatchBuilder and use its methods to create Patch operations.

import {PatchBuilder, LogicalClock} from 'json-joy/lib/json-crdt-patch';

// The clock specifies the patch ID, i.e. the ID of the first operation
// in the patch. The clock is used to generate unique IDs for all operations
// in the patch.
const clock = new LogicalClock(123, 456);
const builder = new PatchBuilder(clock);

Creating a Patch

You can use PatchBuilder methods which generate JSON CRDT Patch methods directly, such as str or insStr, which create a new "str" RGA-String data types and inset a string into it, respectively.

const strId = builder.str();
builder.insStr(strId, strId, 'Hello, world!');

You can use the root helper method to assign a value to the JSON document root LWW-Register.

builder.root(strId);

Now you can retrieve the Patch instance by calling flush.

const patch = builder.flush();

A patch can be printed to terminal in a human-readable format using its built-in toString method.

console.log(patch.toString());
// Patch 123.456!15
// ├─ "str" 123.456
// ├─ "ins_str" 123.457!13, obj = 123.456 { 123.456 ← "Hello, world!" }
// └─ "ins_val" 123.470!1, obj = 0.0, val = 123.456

One can also use various JSON helper methods like json which recursively create the necessary operations to create a JSON document.

builder.root(builder.json({
  foo: 'bar',
  baz: 123,
}));

const patch = builder.flush();
console.log(patch.toString());
// Patch 123.456!8
// ├─ "obj" 123.456
// ├─ "str" 123.457
// ├─ "ins_str" 123.458!3, obj = 123.457 { 123.457 ← "bar" }
// ├─ "con" 123.461 { 123 }
// ├─ "ins_obj" 123.462!1, obj = 123.456
// │   ├─ "foo": 123.457
// │   └─ "baz": 123.461
// └─ "ins_val" 123.463!1, obj = 0.0, val = 123.456

Serializing and Deserializing a Patch

Patch can be serialized into a "binary" format using the build-in toBinary method, and then deserialized back using the fromBinary method.

const buf = patch.toBinary();
const patch2 = Patch.fromBinary(buf);

Also, "verbose" and "compact" serialization formats are supported. See the following modules:

  • json-crdt-patch/codec/verbose module implements "verbose" serialization codec.
  • json-crdt-patch/codec/compact module implements "compact" serialization codec.
  • json-crdt-patch/codec/binary module implements "binary" serialization codec.
